OrientedObjectProgram Start

  • Methods are functions that are specific to a class.

    Every object also has a type, called its class.

    To create new types of data, we implement new classes.

    Class names are conventionally written using the CapWords convention (also called CamelCase because the capital letters in the middle of a name look like humps).
    Method names follow the standard convention of naming functions using lowercased words separated by underscores.

  • The built-in function getattr also returns an attribute for an object by name.

    We can also test whether an object has a named attribute with hasattr.

  • This attribute can still be accessed from any instance of the class.

    >>> class Account:
                                    interest = 0.02            # A class attribute
                                    def __init__(self, account_holder):
                                        self.balance = 0
                                        self.holder = account_holder
                                    # Additional methods would be defined here

    全体访问

    >>> spock_account = Account('Spock')
                                >>> kirk_account = Account('Kirk')
                                >>> spock_account.interest
                                0.02
                                >>> kirk_account.interest
                                0.02

    一改全改

    >>> Account.interest = 0.04
                                >>> spock_account.interest
                                0.04
                                >>> kirk_account.interest
                                0.04
